## Action Items

### Major

- [ ] **Close (X) button does nothing in the Set Edit sheet on mobile**
  - **What:** On a mobile viewport, tapping the X to close the Set Edit sheet has no effect. (Works on desktop.)
  - **Where:** `/log/$sessionId` -> `src/components/session/SetEditSheet.tsx:223-231` (`SetEditSheet`, close button) — the button is nested inside the header `<div>` at `src/components/session/SetEditSheet.tsx:207-211` which has `onPointerDown={onDragStart}`.
  - **Type:** bug
  - **Fix direction:** The header's `onDragStart` (line 134) calls `setPointerCapture(e.pointerId)` on mobile (`max-width: 639px`); that capture swallows the pointerup so the close button's `onClick={requestClose}` never fires. Stop the button from initiating the header drag — add `onPointerDown={(e) => e.stopPropagation()}` to the close button, or move the drag handlers off the full header so they bind only to the grab-handle div (`src/components/session/SetEditSheet.tsx:195-204`). Ensure drag-to-dismiss still works after the change.
  - **Acceptance:** On a mobile/touch viewport, tapping X closes the sheet (or shows the discard-confirm when the set is dirty); drag-to-dismiss via the grab handle still functions.

- [ ] **Drag-handle near-misses trigger Safari pull-to-refresh, reloading the page mid-session**
  - **What:** On mobile, a downward swipe that starts slightly off the sheet's grab handle triggers the browser's native pull-to-refresh and reloads the whole page (losing sheet state) instead of dragging the sheet.
  - **Where:** `/log/$sessionId` -> `src/components/session/SetEditSheet.tsx:195-204` (grab-handle area) and the fixed overlay container at `src/components/session/SetEditSheet.tsx:173`.
  - **Type:** bug
  - **Fix direction:** The grab bar has `touch-action: none`, but touches landing just outside it hit the page/backdrop where default overscroll allows pull-to-refresh. Suppress pull-to-refresh while the sheet is open — set `overscroll-behavior-y: contain` (or `none`) on the overlay container/backdrop (and consider applying it to `html`/`body` while a sheet is mounted), and enlarge the drag-catch zone so near-misses initiate the sheet drag rather than the page gesture. Keep `touch-action: none` on the active drag region.
  - **Acceptance:** On a mobile/touch viewport, a downward swipe starting near (but not exactly on) the grab handle does not trigger browser pull-to-refresh; it drags the sheet or does nothing.

### Minor

- [ ] **Horizontal scrollbar visible on the muscle-group filter chips row (small screens)**
  - **What:** On small screens the muscle-group chips row in the Exercise Library shows a visible horizontal scrollbar track.
  - **Where:** `/exercises` -> `src/routes/exercises/index.tsx:105` (`ExercisesPage`, the chips `<div className="flex gap-1.5 overflow-x-auto pb-1 mb-3 scrollbar-none">`).
  - **Type:** bug (CSS)
  - **Fix direction:** The `scrollbar-none` class is referenced but never defined (not in `src/styles.css`, not a Tailwind v4 built-in), so the scrollbar is never hidden. Define a real utility in `src/styles.css`: `scrollbar-width: none; -ms-overflow-style: none;` plus a `::-webkit-scrollbar { display: none; }` rule scoped to the utility. (Verify whether the Category/Equipment dropdown row at `src/routes/exercises/index.tsx:134` shares the same need.)
  - **Acceptance:** The chips row still scrolls horizontally on narrow screens, but no scrollbar track is visible.
