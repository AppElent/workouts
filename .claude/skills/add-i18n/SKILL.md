# add-i18n candidate capability

This is a candidate capability skill, not a fully implemented stamping skill yet. It exists so web/local agents can recognize the capability without inventing a local-only pattern.

Before implementing anything:

1. Check the Appelent registry and managed capabilities for the project.
2. Do not stamp code yet or assume a final local pattern.
3. Decide whether i18n support belongs as a package, a bootstrap option, or a capability skill.
4. Write and get approval for a dedicated design/plan before implementation.

Implementation should wait until that design is explicit and accepted.
